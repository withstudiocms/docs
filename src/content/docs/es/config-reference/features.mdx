## `robotsTXT`

`robotsTXT` permite al usuario habilitar/deshabilitar y configurar el uso de la integración personalizada `astro-robots-txt` de StudioCMS.

- **Tipo:** `RobotsConfig` | `boolean`
- **Valor por defecto:** `{ policy: [ { userAgent: ['*'], allow: ['/'], disallow: ['/dashboard/'] } ] }`

### Uso

Nota: `robotsTXT` también se puede establecer en `false` para deshabilitarlo.

```ts twoslash {3-11} title="studiocms.config.mjs"
import { defineStudioCMSConfig } from 'studiocms/config';
// ---cut---
export default defineStudioCMSConfig({
  features: {
    robotsTXT: {
      policy: [ 
        { 
          userAgent: ['*'], 
          allow: ['/'], 
          disallow: ['/dashboard/'],
        } 
      ] 
    }
  }
})
```

## `injectQuickActionsMenu`

`injectQuickActionsMenu` permite habilitar o deshabilitar el menú de acciones rápidas que permite un acceso fácil al panel de control (dashboard) mientras se está conectado en páginas que no son del panel.

- **Tipo:** `boolean`
- **Valor por defecto:** `true`

### Uso

```ts twoslash {3} title="studiocms.config.mjs"
import { defineStudioCMSConfig } from 'studiocms/config';
// ---cut---
export default defineStudioCMSConfig({
  features: {
    injectQuickActionsMenu: true
  }
})
```

## `sdk`

`sdk` puede ser un booleano o un objeto que contiene la configuración de caché. Si es un booleano, su valor por defecto es `true` y se transforma en un objeto con la configuración de caché predeterminada.

### Uso

```ts twoslash {3} title="studiocms.config.mjs"
import { defineStudioCMSConfig } from 'studiocms/config';
// ---cut---
export default defineStudioCMSConfig({
  features: {
    sdk: {}
  }
})
```

### `cacheConfig`

`cacheConfig` es un objeto que se utiliza para configurar la caché del SDK.

- **Tipo:** `boolean` | `{ lifetime?: string | undefined; }` | `undefined`
- **Valor por defecto:** `{ lifetime: '5m' }`

#### Uso

```ts twoslash {5-7} title="studiocms.config.mjs"
import { defineStudioCMSConfig } from 'studiocms/config';
// ---cut---
export default defineStudioCMSConfig({
  features: {
    sdk: {
      // VALOR POR DEFECTO - Esto utiliza la configuración de caché predeterminada.
      cacheConfig: {
          lifetime: '5m',
      }, 
    }
  }
})
```

### `dashboardEnabled`

`dashboardEnabled` permite al usuario habilitar o deshabilitar el panel de control de StudioCMS, pero aún así proporciona todos los asistentes y utilidades para aquellos que personalizan su configuración. Al hacerlo, se deshabilitará el panel de control y deberás gestionar tu contenido a través de tu base de datos.

#### Uso

```ts twoslash {4} title="studiocms.config.mjs"
import { defineStudioCMSConfig } from 'studiocms/config';
// ---cut---
export default defineStudioCMSConfig({
  features: {
    dashboardConfig: {
      dashboardEnabled: true,
    }
  }
})
```

### `providers`

Permite habilitar o deshabilitar los Proveedores de Autenticación

El siguiente proveedor se puede configurar desde aquí: `usernameAndPassword`.

#### Uso

```ts twoslash {4-9} title="studiocms.config.mjs"
import { defineStudioCMSConfig } from 'studiocms/config';
// ---cut---

  features: {
    authConfig: {
      providers: {
        usernameAndPassword: true,
        usernameAndPasswordConfig: {
          allowUserRegistration: true
        }
      }
    }
  }
})
```

### `demoMode`

Activa el modo de demostración para el sitio. Si se establece como un objeto, el sitio estará en modo de demostración y el usuario podrá iniciar sesión con el nombre de usuario y contraseña proporcionados.

#### Uso

```ts twoslash {4-7} title="studiocms.config.mjs"
import { defineStudioCMSConfig } from 'studiocms/config';
// ---cut---
export default defineStudioCMSConfig({
  features: {
    developerConfig: {
      demoMode: {
        username: 'usuario', // Ejemplo: 'foo'
        password: 'contraseña' // Ejemplo: 'bar'
      }
    }
  }
})
```
